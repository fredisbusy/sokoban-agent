# Project Guidance

## 작업 시작

- 먼저 `docs/PROJECT.md`에서 목표와 범위를, `TODO.md`에서 현재 우선순위와
  완료 조건을 확인한다.
- 사용자의 명시적 요청이 항상 TODO보다 우선한다. 다음 작업을 자율적으로
  선택해야 할 때만 `TODO.md`의 `지금`에서 첫 미완료 항목을 고른다.
- 완료한 작업은 검증이 통과한 뒤에만 `TODO.md`에 반영한다.

## 필수 검증

- 테스트: `uv run pytest`
- 린트: `uv run ruff check .`
- 타입 검사: `uv run mypy`

## 커밋 메시지

- 모든 커밋은 Conventional Commits 형식을 사용한다.
- 형식은 `<type>(<scope>): <한글 설명>` 또는 `<type>: <한글 설명>`이다.
- `type`은 영문 소문자로 작성하고, 설명은 간결한 한글로 작성한다.
- 주로 `feat`, `fix`, `docs`, `test`, `refactor`, `perf`, `build`, `ci`,
  `chore`를 사용한다.
- 예시: `chore: 프로젝트 초기 구조 설정`
- 호환성이 깨지는 변경은 제목에 `!`를 붙이고 본문에 `BREAKING CHANGE:`를
  기록한다.

## 작업 완료

- 요청받은 작업을 구현하고 관련 검증을 마친 뒤에는 변경 사항을 반드시
  커밋한다.
- 커밋에는 해당 작업과 직접 관련된 파일만 포함하고, 관련 없는 변경은
  스테이징하지 않는다.
- 사용자가 푸시를 요청하지 않았다면 커밋까지만 수행하고 자동으로 푸시하지
  않는다.
